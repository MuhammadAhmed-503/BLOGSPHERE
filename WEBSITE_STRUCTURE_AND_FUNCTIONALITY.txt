BLOGSPHERE - WEBSITE STRUCTURE AND FUNCTIONALITY

Note:
- This document lists folders and files in the source workspace.
- Dependency/build folders like node_modules and dist are not documented as source code.
- Each item has one primary functionality.

ROOT FOLDERS
- .vscode/ : Editor workspace settings for VS Code.
- api/ : Vercel serverless API entrypoints.
- backend/ : Express + MongoDB backend API and business logic.
- frontend/ : Vite + React frontend web application.

ROOT FILES
- .env.example : Example environment variables for root-level setup.
- .env.local : Local environment variables for development.
- .eslintrc.js : Linting configuration for code quality rules.
- .gitignore : Git ignore rules to exclude generated/secret files.
- .vercelignore : Vercel ignore rules for deployment upload filtering.
- Blog Saas.code-workspace : VS Code multi-root workspace definition.
- package-lock.json : Exact dependency lockfile for reproducible installs.
- package.json : Root scripts and shared dependency definitions.
- README.md : Project documentation and setup instructions.
- vercel.json : Vercel deployment/build/route configuration.
- WEBSITE_STRUCTURE_AND_FUNCTIONALITY.txt : Structure and functionality reference document.

VSCODE
- .vscode/settings.json : Local editor settings for this workspace.

API (VERCEL)
- api/[...path].ts : Catch-all serverless handler that boots backend app and serves API routes.

BACKEND FOLDER
- backend/.env.example : Backend-specific environment variable template.
- backend/package.json : Backend package scripts and dependency manifest.
- backend/tsconfig.json : TypeScript compiler config for backend.

BACKEND SOURCE
- backend/src/app.ts : Creates and configures the Express app, middleware, and routes.
- backend/src/index.ts : Starts the backend server process for local runtime.
- backend/src/validators.ts : Shared request validation schemas/rules.

BACKEND CONFIG
- backend/src/config/database.ts : MongoDB connection initialization and handling.
- backend/src/config/env.ts : Environment variable loading, parsing, and health checks.

BACKEND DATA
- backend/src/data/seedPosts.ts : Seed post dataset used for initial content bootstrap.

BACKEND MIDDLEWARE
- backend/src/middleware/auth.ts : JWT/auth role protection middleware.
- backend/src/middleware/errorHandler.ts : Centralized error and not-found handlers.
- backend/src/middleware/validate.ts : Request payload validation middleware wrapper.

BACKEND MODELS
- backend/src/models/Comment.ts : Mongoose model/schema for blog comments.
- backend/src/models/ContactMessage.ts : Mongoose model/schema for contact messages.
- backend/src/models/Post.ts : Mongoose model/schema for blog posts.
- backend/src/models/PushSubscription.ts : Mongoose model/schema for web push subscribers.
- backend/src/models/SiteSetting.ts : Mongoose model/schema for site configuration.
- backend/src/models/Subscription.ts : Mongoose model/schema for newsletter subscriptions.
- backend/src/models/User.ts : Mongoose model/schema for users/admin accounts.

BACKEND ROUTES
- backend/src/routes/admin.ts : Admin CRUD/API endpoints for protected operations.
- backend/src/routes/auth.ts : Authentication endpoints (login/register/token flows).
- backend/src/routes/contact.ts : Contact form submission endpoints.
- backend/src/routes/health.ts : Health-check endpoints for uptime monitoring.
- backend/src/routes/public.ts : Public blog/content endpoints for frontend.
- backend/src/routes/push.ts : Web push subscription/notification endpoints.
- backend/src/routes/uploads.ts : File upload endpoints (media handling).

BACKEND SCRIPTS
- backend/src/scripts/resetViews.ts : Script to reset post view counters/analytics values.
- backend/src/scripts/seed.ts : Script to seed database with starter content.

BACKEND SERVICES
- backend/src/services/bootstrap.ts : Startup bootstrap tasks for initial app state.
- backend/src/services/cloudinary.ts : Cloudinary media upload/service integration.
- backend/src/services/email.ts : SMTP email sending service logic.
- backend/src/services/push.ts : Web push notification sending service.

BACKEND TYPES
- backend/src/types/express.d.ts : Express type augmentation for custom request fields.

BACKEND UTILS
- backend/src/utils/appError.ts : Custom application error class/helpers.
- backend/src/utils/asyncHandler.ts : Async route wrapper for error propagation.
- backend/src/utils/jwt.ts : JWT creation/verification utility functions.
- backend/src/utils/pagination.ts : Pagination helpers for list endpoints.
- backend/src/utils/serialize.ts : Serialization helpers for API response shaping.
- backend/src/utils/slug.ts : Slug generation/normalization utility functions.

FRONTEND FOLDER
- frontend/.env.example : Frontend environment variable template.
- frontend/.env.local : Local frontend environment variables.
- frontend/index.html : Vite HTML shell for app mounting.
- frontend/postcss.config.js : PostCSS pipeline configuration.
- frontend/tailwind.config.js : Tailwind theme/content configuration.
- frontend/tsconfig.json : TypeScript compiler settings for frontend.
- frontend/tsconfig.tsbuildinfo : TypeScript incremental build cache metadata.
- frontend/vite-env.d.ts : Vite TypeScript environment type definitions.
- frontend/vite.config.ts : Vite build/dev server configuration.

FRONTEND PUBLIC FOLDER
